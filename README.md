## About The Project

ERC721M is an NFT contract template designed to align NFT collections with Remilia's vision for Network Spirituality by deepening NFT collection liquidity using mint fees via an [AlignmentVault](https://github.com/Zodomo/AlignmentVault).
The template allows the deployer to receive up to 95% of the mint fees, while the remainder is dedicated to deepening an NFT's NFTX liquidity. The deployer can align between 5-100% of mint fees.
All aligned funds are directed towards deepening NFTX liquidity for a particular NFT collection and are locked forever. Yield generated by that liquidity can still be claimed by the AlignmentVault owner.

## Getting Started

ERC721M was designed using Foundry, so we recommend familiarizing yourself with that if required.

### Prerequisites

- Foundry
  ```sh
  curl -L https://foundry.paradigm.xyz | bash
  foundryup
  ```

### Installation

1. Set up your NFT project using Foundry
   ```sh
   forge init ProjectName
   ```
2. Install ERC721M
   ```sh
   forge install zodomo/ERC721M --no-commit
   ```
3. Import ERC721M<br />
   Add the following above the beginning of your project's primary contract
   ```solidity
   import "ERC721M/ERC721M.sol";
   ```
4. Inherit the module<br />
   Add the following to the contract declaration
   ```solidity
   contract ProjectName is ERC721M {}
   ```
5. Utilize each function as required<br />
   A deeper understanding of Solady is required to continue beyond this point

## Contract Initialization

- **initialize:** Initializes the contract with various parameters including name, symbol, base URI, max supply, royalty, mint allocation, owner, aligned NFT, price, NFTX vault ID and deployment salt. This function should be called immediately after deployment.

<p align="right">(<a href="#readme-top">back to top</a>)</p>

## View and Metadata Functions

- **name, symbol, baseURI, contractURI:** View functions to get the collection's name, symbol, base URI and contract URI.
- **tokenURI:** Returns the token URI for a given token ID.
- **totalSupply:** Returns the current total supply of tokens.
- **supportsInterface:** Checks if the contract supports a given interface.
- **getBlacklist, getCustomMintListIds:** Get the list of blacklisted addresses and custom mint list IDs.

<p align="right">(<a href="#readme-top">back to top</a>)</p>

## Mint Logic

- **\_mint:** Internal function to mint tokens. Implements mint funds alignment and blacklist checks.
- **mint:** Various overloads of the public mint function to support different scenarios (single mint, batch mint, referral, custom allocation).
- **customMint:** Whitelisted mint using merkle proofs.

<p align="right">(<a href="#readme-top">back to top</a>)</p>

## Custom Mint Functions

- **setCustomMint:** Set up a custom mint list using merkle trees.
- **disableCustomMint:** Disable a custom mint list.
- **reenableCustomMint:** Re-enable a previously disabled custom mint list.
- **repriceCustomMint:** Change the price for a custom mint list.
- **nukeCustomMint:** Completely delete a custom mint list.

<p align="right">(<a href="#readme-top">back to top</a>)</p>

## Owner Functions

- **setReferralFee:** Set the referral fee.
- **setBaseURI, lockURI:** Set the base URI for token metadata and optionally lock it permanently.
- **setPrice:** Set the price for minting.
- **setRoyalties, setRoyaltiesForId:** Set royalties for the whole collection or for specific token IDs.
- **disableRoyalties:** Permanently disable royalties.
- **setBlacklist:** Configure the blacklist.
- **openMint:** Open the minting.
- **increaseAlignment:** Increase the mint alignment allocation. This is a one-way operation.
- **decreaseSupply:** Decrease the maximum supply. This is a one-way operation.
- **withdrawFunds:** Withdraw non-allocated mint funds.

<p align="right">(<a href="#readme-top">back to top</a>)</p>

## Asset Handling

- **rescueERC20, rescueERC721:** Rescue non-aligned tokens and NFTs from the contract. Aligned assets are forwarded to the vault.
- **onERC721Received:** Forward received aligned NFTs to the vault.
- **receive, fallback:** Handle received Ether. If minting is open, it mints tokens, else it forwards the funds to the vault.

## AlignmentVault

A testnet AlignmentVaultFactory has been deployed to <a href="https://sepolia.etherscan.io/address/0x7c1A6B4B373E70730c52dfCB2e0A67E7591d4AAa">Etherscan</a>. Please use this in your contracts or manually to deploy AlignmentVaults, as it will be cheaper and will utilize the implementation already deployed on-chain. The factory is designed to return the current implementation address by calling the implementation() function.
